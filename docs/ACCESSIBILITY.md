# Accessibility

## Draft 1 Requirements

- Mobile-first layout.
- Large touch targets for search and filters.
- High contrast text and buttons.
- Keyboard-focusable controls.
- Map plus list, never map-only.
- Plain language summaries before source details.

## Future Work

- Hindi-first mode plus major state languages.
- Voice readout for result summaries and complaint routing.
- Reduced-data mode for low-end phones and slow networks.
- Screen-reader QA against the main user journeys.
- Formal audit against WCAG-style best practices and Indian government app guidance.
